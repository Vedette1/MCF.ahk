# MCF (MCode Forge)

MCF (MCode Forge) — это автономный COFF-линковщик, реализованный на AutoHotkey v2 и предназначенный для самой платформы AutoHotkey v2.

![Screenshot](https://github.com/Vedette1/MCF.ahk/blob/main/screenshot1.png)

## О проекте

Основная цель проекта — это простота сборки Mcode из исходников на C. Пишите обыденный C-код и в один клик кнопки получите готовый Mcode.


## Архитектура и принцип работы линковщика (Класс COFF)

Ядром проекта является кастомный линковщик (класс `COFF`), реализованный на чистом AutoHotkey v2. Он выполняет парсинг и сборку объектных файлов формата COFF (Common Object File Format), генерируемых компиляторами (GCC/Clang, MSVC), и преобразует их в готовый машинный код (`MCode`) без привлечения стороннего ПО.

1. **Парсинг структуры заголовков:** Линковщик считывает заголовки файла (`IMAGE_FILE_HEADER`), таблицы секций (`IMAGE_SECTION_HEADER`) и таблицу символов (`IMAGE_SYMBOL`). На этом этапе фильтруются ненужные для `MCode` служебные секции (Линковщик по умолчанию пропускает секции со следующими характеристиками: `IMAGE_SCN_LNK_REMOVE`, `IMAGE_SCN_LNK_INFO`, `IMAGE_SCN_MEM_DISCARDABLE`), и кроме того вы так же можете указать секции (по именам), которые будут игнорироваться при линковке (по умолчанию это: `.pdata`, `.xdata`, `.rdata$zzz`).
2. **Формирование Layout (План памяти):** Секции выстраиваются друг за другом в единый непрерывный буфер с учетом выравнивания памяти из характеристик (`IMAGE_SECTION_HEADER.Characteristics`), и рассчитываются новые смещения для каждой секции.
3. **Разрешение релокаций (Адресный патчинг):** * **Внутренние релокации:** Происходит обход таблицы релокаций (`IMAGE_RELOCATION`). Смещения инструкций внутри кода автоматически корректируются (все поддерживающиеся типы релокаций смотрите в методе `ApplyRelocation`) для корректного межсекционного взаимодействия переменных и функций в результирующем буфере.
   * **Внешние вызовы (Импорт):** Линковщик находит внешние символы (например, вызовы функций WinAPI через `__imp_`). Вместо генерации полноценной классической PE-таблицы импорта (`IAT`), проект использует оптимизированное решение — формируется кастомный компактный **IAT-хвост** (выглядит это примерно так: `|msvcrt:isspace:51:4` - [`|dllName:funcName:reloc:relocType|...`]), содержащий метаданные о DLL, именах функций и смещениях патчинга.
4. **Генерация таблицы символов:** Линковщик экспортирует карту смещений (функций, глобальных переменных и секций), что позволяет узнать смещение точки входа в конкретных C/C++ функций внутри полученного блока машинного кода.


## Графический интерфейс (GUI)

Проект оснащен удобным графическим интерфейсом, который спроектирован как **микро-IDE** для удобства написания кода, если у вас нет нормального IDE.

### Ключевые особенности GUI:
* **Интегрированная мини-IDE:** Текстовая область поддерживает подсветку синтаксиса для исходного кода на C/C++, а также для дизассемблированного вывода `objdump` (в данный момент поддерживается только синтаксис intel).
* **Полная автономность (Zero Dependencies):** Работа GUI и встроенного линковщика **не привязана** жестко к компилятору или утилите `objdump`. Они являются лишь опциональными вспомогательными инструментами для автоматизации компиляции, и отладочной информации ввиде дизассемблере.
* **Сборка из готовых файлов:** Если у вас нет установленного в системе компилятора, вы можете просто указать путь к любому готовому объектному файлу (`.o` / `.obj`), и встроенный линковщик успешно соберет `MCode` без каких-либо внешних зависимостей.
* **Гибкий экспорт:** Результат линковки можно в один клик получить в форматах `HEX`, `Base64` или в сжатом виде (`Compressed Base64` через встроенные алгоритмы компрессии `ntdll.dll`).

## Example
```ahk
/**
#include <windows.h>
int msgBox() {
	return MessageBox(0, "Hello Mcode", "Title", 0);
}
*/
; compiler : TDM-GCC 10.3.0
; flags    : -m64 -O2 -fPIC -fno-stack-protector -fno-plt -fno-asynchronous-unwind-tables

offset := Map()
offset["msgBox"] := 0x0 ; 2
offset[".text"] := 0x0 ; 4 [dispensable]
offset[".data"] := 0x20 ; 6 [dispensable]
offset[".bss"] := 0x20 ; 8 [dispensable]
offset[".rdata"] := 0x20 ; 10 [dispensable]

ptr := GetMcodePtr("4531c94c8d0516000000488d151500000031c9ff2500000000909090909090905469746c650048656c6c6f204d636f6465000000000000000000000000000000|User32:MessageBoxA:21:4") ; Copy the GetMcodePtr function from the GUI.
DllCall(ptr, "int")
```

## Текущие ограничения

* **CRT (C Runtime):** Линковщик не поддерживает C Runtime Library (CRT). Код, использующий стандартные функции C, такие как `printf`, в данный момент не может быть обработан корректно. Причина заключается в том, что CRT требует линковки со статическими библиотеками, парсер для которых пока не реализован. Например, если вы видите **странные** имена функций, которых явно нет в исходном коде (`__acrt_iob_func`, `__mingw_vfprintf` и подобные), то ваш `Mcode` не будет работать.
* **SEH (Structured Exception Handling):** Обработка исключений (`try / catch`) в x64 требует регистрации таблиц исключений (`.pdata` и `.xdata`) в ядре Windows через `RtlAddFunctionTable`. Я не имею ни малейшего понятия, как это сделать, хотя я даже не пробовал с этим разобраться.
